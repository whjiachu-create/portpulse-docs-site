# Postman

Use this guide to import the **PortPulse OpenAPI**, set a header-based API key, and send your first request.

## What you’ll need

- **Base URL**: `https://api.useportpulse.com`  
- **Auth header**: `X-API-Key: <YOUR_KEY>`  
- **Formats**: JSON & CSV (CSV supports strong **ETag/304**)

---

## 1) Import the OpenAPI

**Option A — URL (recommended)**

1. Open Postman → **Import** → **Link**  
2. Paste the OpenAPI URL:
https://api.useportpulse.com/openapi.json
3. Click **Import**. Postman will create a Collection (and optionally an API entry).

**Option B — File**

If your network blocks external URLs, use the local schema at `docs-site/static/openapi.json` (generated during `pnpm build`). In Postman: **Import → Files** and select that file.

> Note: If you imported an **API** instead of a **Collection**, also generate a collection from it, or re-import choosing **Collection**.

---

## 2) Set variables / headers

Create either a **Collection variable** or an **Environment**:

- `baseUrl` → `https://api.useportpulse.com` → Use in URL as `&#123;&#123;baseUrl&#125;&#125;`  
- `X-API-Key` → `DEMO_KEY` (or your live key)

Then add a header in the request:
Key: X-API-Key
Val: &#123;&#123;X-API-Key&#125;&#125;
This keeps your key out of the URL and cURL history.

---

## 3) Send your first request

Choose endpoint ``GET /v1/ports/`{\1}`/trend`` (or create a new request):
GET &#123;&#123;baseUrl&#125;&#125;/v1/ports/USLAX/trend?window=14d
Header: X-API-Key: &#123;&#123;X-API-Key&#125;&#125;
**Expected:** HTTP 200 with a JSON array like:

```json
[
  {"date":"2025-08-01","congestion_score":0.45, "as_of":"2025-08-01T12:00:00Z"}
]
Try CSV by adding &format=csv.

4) Optional: ETag / 304 workflow (CSV)

Save the ETag from a CSV response and perform a conditional GET.

Tests tab (on the CSV request):
const etag = pm.response.headers.get('etag');
if (etag) pm.environment.set('pp_etag', etag);
Next request header:
If-None-Match: &#123;&#123;pp_etag&#125;&#125;
If data didn’t change, you’ll get 304 Not Modified.

Troubleshooting
	•	401 / 403 → Check X-API-Key and key scope. See Authentication.
	•	404 → Verify path and UN/LOCODE. See API Endpoints.
	•	429 → You’re rate limited. See Rate limits.
	•	Error body → Unified JSON: {code, message, request_id, hint}. See Errors.
